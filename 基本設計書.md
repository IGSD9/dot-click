## この書の見方

開発は **Step 1 → 2 → 3 → 4** の順で進める。  
各 Step に「何を作るか」「完成の目安」を書いてある。コードレベルの仕様は詳細設計書を見る。

```
Step 0  全体像を理解する（本セクション）
  ↓
Step 1  環境構築・DB
  ↓
Step 2  ゲーム本体（矢を刺すコア）
  ↓
Step 3  ログイン・スコア保存・ランキング
  ↓
Step 4  見た目・PWA（アプリ化）※最終形。当面はブラウザで動けば OK
```

### 開発方針（重要）

| 項目 | 方針 |
|------|------|
| **当面のゴール** | **ブラウザで動くこと**（`npm run dev` → PC / スマホブラウザでプレイ） |
| **最終形** | PWA（Step 4）でホーム画面追加 → アプリライクに起動 |
| **対応デバイス** | PC とスマホの **両方** を Step 2 から意識して作る |

**PC（マウス・キーボード）**

| 操作 | 入力 |
|------|------|
| 矢を射つ | クリック / スペースキー |
| 画面 | 横長・縦長どちらでもレイアウトが崩れない |

**スマホ（タッチ）**

| 操作 | 入力 |
|------|------|
| 矢を射つ | 画面タップ（どこでも OK） |
| 画面 | 縦向き優先、`100dvh` でアドレスバー込みの高さ対応 |
| 誤操作防止 | スクロール・ズーム・ダブルタップズームを無効化 |

Step 2 の完成チェックに **PC とスマホ両方で矢が射てること** を必須とする。  
PWA / ホーム画面追加は Step 4（後回しで OK）。

```

## Step 0｜まず全体像

### 0-1. 何を作るか

HitArrow 型の Web ゲーム。  
**回転する的に矢を刺す。既存の矢に当たったらゲームオーバー。**

| やること | 内容 |
|---------|------|
| 操作 | タップ / クリック / スペースで矢を射つ |
| クリア | ステージごとの規定本数を全部刺す |
| 失敗 | 刺さった矢同士がぶつかったら GO |
| 難しくなる | 回転が速くなる・逆回転・最初から矢が刺さっている |

### 0-2. 使う技術（一覧）

| 用途 | 技術 |
|------|------|
| アプリ本体 | Next.js（App Router） |
| DB・認証 | Supabase |
| DB 操作 | Prisma |
| 公開 | Vercel（東京リージョン） |
| 見た目 | Tailwind CSS |
| スマホアプリ化 | PWA |

### 0-3. システムのざっくり構成

```
[ユーザー]
    ↓ ブラウザ / PWA
[Next.js]
  ├─ ゲーム画面（Canvas）     ← Step 2
  ├─ ログイン・ランキング     ← Step 3
  └─ Server Actions          ← Step 3（DB 保存）
    ↓
[Supabase]
  ├─ PostgreSQL（User, Score）
  └─ Auth（ログイン）
```

### 0-4. 画面一覧

| 画面 | パス | いつ作る |
|------|------|---------|
| トップ | `/` | Step 4 |
| ゲーム | `/game` | Step 2 |
| ランキング | `/leaderboard` | Step 3 |
| ログイン | `/auth/login` | Step 3 |

### 0-5. データの持ち方（概要）

| データ | どこに保存 | いつ |
|--------|-----------|------|
| プレイ中の状態 | ブラウザ内（React） | ゲーム中 |
| ログイン情報 | Supabase Auth | ログイン時 |
| 最高スコア | PostgreSQL | ゲームオーバー時 |

```
User（ユーザー）
  └─ Score（1人1レコード）
       ├─ maxStage   … 最高到達ステージ
       └─ totalScore … ハイスコア
```

## Step 2｜ゲーム本体

> **ゴール:** ログインなしで矢を刺してプレイできる

### 2-1. やること

| # | 作業 | 成果物 |
|---|------|--------|
| 1 | Canvas で的を描画・回転 | `GameCanvas.tsx` |
| 2 | タップで矢を射つ | 入力処理 |
| 3 | 衝突判定 | `collision.ts` |
| 4 | ステージ進行 | `stages.ts` |
| 5 | HUD 表示 | ステージ数・残り矢・連続クリア |

### 2-2. ゲームの流れ

```
ステージ開始
  ↓
[idle] タップ待ち
  ↓ タップ
[shooting] 矢が上に飛ぶ
  ↓ 的に到達
  ├─ 既存の矢と近い → [gameover]
  └─ OK → 矢が刺さる → 残り本数 -1
       ↓
       ├─ 残り 0 → [clear] → 次ステージ
       └─ 残りあり → [idle] に戻る
```

### 2-3. コードの置き場所

| 処理 | ファイル | 実行場所 |
|------|---------|---------|
| 描画・ループ | `components/game/GameCanvas.tsx` | ブラウザ（Client） |
| 衝突判定 | `lib/game/collision.ts` | ブラウザ |
| ステージ設定 | `lib/game/stages.ts` | ブラウザ |

**ポイント:** ゲーム処理は全部ブラウザ側。DB は触らない。

### 2-4. 完成チェック

- [ ] 的が回転している
- [ ] **PC:** クリック / スペースで矢が飛ぶ
- [ ] **スマホ:** タップで矢が飛ぶ（スクロールしない）
- [ ] PC・スマホ両方でレイアウトが崩れない
- [ ] 既存の矢に当たると GAME OVER
- [ ] 規定本数刺すと次ステージへ
- [ ] ステージが上がると難しくなる

→ 詳細: [詳細設計書 Step 2](./詳細設計書.md#step-2ゲーム本体)

## Step 3｜ログイン・スコア・ランキング

> **ゴール:** ログインしてスコアが保存され、ランキングが見られる

### 3-1. やること

| # | 作業 | 成果物 |
|---|------|--------|
| 1 | Supabase Auth 設定 | ログイン画面 |
| 2 | 初回ログイン時に User 作成 | `actions/user.ts` |
| 3 | ゲームオーバー時にスコア保存 | `actions/score.ts` |
| 4 | ランキング画面 | `/leaderboard` |

### 3-2. ユーザーの流れ

**未ログイン**
```
ゲーム → GO → 「ログインして保存」表示
```

**ログイン済**
```
ゲーム → GO → 自動でスコア保存 → ランキングで確認
```

### 3-3. Server Actions 一覧

| Action | いつ呼ぶ | 何をする |
|--------|---------|---------|
| `saveScore` | ゲームオーバー | ハイスコア更新 |
| `getLeaderboard` | ランキング表示 | TOP 50 取得 |
| `getMyScore` | プロフィール等 | 自分のベスト |

### 3-4. スコアの計算（概要）

```
totalScore = 到達ステージ × 1000 + 連続クリアボーナス
連続クリアボーナス = min(連続数, 50) × 10
```

過去の記録より高いときだけ DB を更新する。

### 3-5. 完成チェック

- [ ] メール or ソーシャルでログインできる
- [ ] ゲームオーバー後、スコアが DB に保存される
- [ ] ランキングに自分の名前が出る
- [ ] 未ログインでもゲームはプレイできる

→ 詳細: [詳細設計書 Step 3](./詳細設計書.md#step-3ログインスコアランキング)

## 付録

### A. 非機能要件（目標値）

| 項目 | 目標 |
|------|------|
| 操作の反応 | タップ後 1 フレーム以内 |
| 描画 | 60fps |
| API 応答 | 200ms 以内 |

### B. セキュリティ方針

| 項目 | 方針 |
|------|------|
| 通信 | HTTPS のみ |
| ログイン | Supabase Auth |
| スコア保存 | ログイン必須、Server Actions で検証 |

### C. 用語集

| 用語 | 意味 |
|------|------|
| ターゲット | 中央で回る円形の的 |
| 障害矢 | ステージ開始時から刺さっている矢（赤色） |
| maxStage | 最高到達ステージ |
| totalScore | ハイスコア |
| PWA | ホーム画面に追加できる Web アプリ |

### D. リスクと対策

| リスク | 対策 |
|--------|------|
| 操作がもっさり | requestAnimationFrame、描画を最小限に |
| 衝突判定がズレる | 許容角度を調整、テストを書く |
| ログインと DB の User が不一致 | 初回ログイン時に自動作成 |

### E. 改訂履歴

|------|------|------|

### F. Git でコードを保存する（初めての方向け）

開発の区切りごとに Git で保存する。クラウド（GitHub 等）に **push** すれば PC 故障時も復元できる。

**基本の流れ**

```
1. コードを変更する
2. git add .          … 保存対象を選ぶ
3. git commit         … ローカルに記録（セーブポイント）
4. git push           … クラウドにアップロード
```

**初回だけ（GitHub に新規リポジトリを作る場合）**

```bash
# GitHub に hitarrow リポジトリを作成後
git remote add origin https://github.com/YOUR_NAME/hitarrow.git
git push -u origin main
```

**コミットしてはいけないもの**

| ファイル | 理由 |
|---------|------|
| `.env.local` | DB パスワード・API キーが入る |
| `node_modules/` | `npm install` で再生成できる |

`.env.example` はテンプレートなのでコミット OK。

**推奨タイミング**

- Step 完了ごと（Step 1 終了、ゲーム動いた、ログイン実装など）
- 動く状態になったとき（「ここまで戻したい」地点）
